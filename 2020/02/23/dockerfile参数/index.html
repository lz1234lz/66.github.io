dockerfile参数
更新于 2020-03-28 docker

Dockerfile

Dockerfile 是一个用来构建镜像的文本文件，文本内容包含了一条条构建镜像所需的指令和说明

要求

  • Dockerfile文件名首字母必须大写
  • Dockerfile整体就两类语句组成，注释和指令（#进行注释）
  • Dockerfile指令不区分大小写
  • Dockerfile中第一个非注释行必须是FROM指令，用来指定制作当前镜像依据的是哪个基础镜像
  • Dockerfile中指令按顺序从上至下依次执行

Dockerfile指令

  • FROM：FROM指令必须为Dockerfile文件开篇的第一个非注释行，用于指定构建镜像所使用的基础镜像，后续的指令运行都要依靠此基础镜像所提供的的环境，格式为FROM <Repository>[:<Tag>]或FROM <Repository>@<Digest>，例如FROM centos

  • MAINTAINER：☞改镜像维护者信息

  • WORKDIR：指以后命令的工作目录，直到遇到下一个WORKDIR
    注意：当设置为 WORKDIR a WORKDIR b链接时，工作目录将为/a/b

  • ENV: 为镜像定义所需的环境变量，并可被ENV指令后面的其它指令所调用。调用格式为$variable_name或者${variable_name}

  • ARG: ARG命令同EVN类似，也是指定一个变量，但不同的是，ENV指令配合-e参数可以在docker run过程中传参，而使用ARG指令配合–build-arg参数可以在docker build过程中传参

  • USER: 用于指定docker build过程中任何RUN、CMD等指令的用户名或者UID

  • RUN: 用于指定docker build过程中运行的程序，可以是任何命令,RUN命令必须在FROM后才能使用，使用形式为RUN command或为RUN ["executable", "param1", "param2"]

  • CMD: 指定启动容器的默认要运行的程序，也就是PID为1的进程命令，且其运行结束后容器也会终止。如果不指定，默认是bash。

    1. CMD指令会被docker run覆盖
    2. 多个CMD指令只有最后一个CMD指令才会生效
    3. RUN指令运行在docker build的过程中，而CMD指令运行在基于新镜像启动容器（docker run）时。

  • ENTRYPOINT: 类似CMD指令的功能，用于为容器指定默认运行程序

    1. Dockerfile中可以存在多个ENTRYPOINT指令，但仅最后一个生效
    2. ENTRYPOINT启动的程序不会被docker run命令行指定的参数所覆盖，而且这些命令行参数会被当做参数传递给ENTRYPOINT指令指定的程序
    3. –entrypoint选项的参数可覆盖ENTRYPOINT指定的默认程序

  • COPY: 复制宿主机上的文件到目标镜像中. COPY <src>... <dest>

    1. <src>必须是build上下文中的目录，不能是其父目录中的文件
    2. 如果<src>是目录，则其内部的文件或则子目录会被递归复制，但<src>目录本身不会被复制
    3. 如果指定了多个<src>，或者<src>中使用通配符，则<dest>必须是一个目录，且必须以 / 结尾
    4. 如果<dest>事先不存在，它将会被自动创建，包括其父目录路径

  • ADD: ADD指令跟COPY类似，不过它还支持使用tar文件和URL路径.

    1. 当拷贝的源文件是tar文件时，会自动展开为一个目录并拷贝进新的镜像中；然而通过URL获取到的tar文件不会自动展开
    2. 主机可以联网的情况下，docker build可以将网络上的某文件引用下载并打包到新的镜像中

  • VOLUME: 用于在镜像中创建一个挂载点目录。只能指定容器内的路径，不能指定宿主机的路径。

  • EXPOSE: 指定容器中待暴露的端口。比如容器提供的是一个http服务且需要对外提供访问，那就需要指定待暴露80端口，然后在使用此镜像启动容器时搭配 -P 的参数才能将待暴露的状态转换为真正暴露的状态，转换的同时80也会转换成一个随机端口，跟 -p :80一个意思

    1. ENV <key> <value> 一次只能定义一个变量，添加多个值，key后面的值都会被认为变量
    2. ENV <key>=<value> 一次可以定义多个变量

  • ONBUILD: 指令在docker build时是不会执行，构建完的镜像在被另一个Dockerfile文件中FROM指令所引用的时才会触发执行
