docker 镜像操作指令

查看已经拉取的镜像

docker images

搜索镜像

NAME: 镜像仓库源的名称
DESCRIPTION: 镜像的描述
OFFICIAL: 是否 docker 官方发布
STARS: 表示点赞、喜欢的数量。
AUTOMATED: 自动构建。
docker search [OPTIONS] container

条件搜索--filter

-f, –filter filter 过滤条件当前支持

* stars 星星数量
* is-automated 是否为自动构建
* is-official 是否为官方镜像

–format string 自己构建的输出模板
–limit int 搜索结果数量（默认25）
–no-trunc 不截断输出

下载镜像docker pull

当然国内因为一些原因，镜像拉取速度可能会有点慢，可以通过以下方法修改镜像拉取源
可供选取的镜像源

  • Docker 官方中国区 https://registry.docker-cn.com
  • 网易 http://hub-mirror.c.163.com
  • ustc https://docker.mirrors.ustc.edu.cn

镜像源修改方法
直接设置 –registry-mirror 参数，仅对当前的命令有效
docker run hello-world --registry-mirror=https://docker.mirrors.ustc.edu.cn
修改 /etc/default/docker，加入 DOCKER_OPTS=”镜像地址”，可以有多个
DOCKER_OPTS="--registry-mirror=https://docker.mirrors.ustc.edu.cn"
支持 systemctl 的系统，通过 sudo systemctl edit docker.service，会生成 etc/systemd/system/docker.service.d/override.conf 覆盖默认的参数，在该文件中加入如下内容
[Service]
ExecStart=/usr/bin/docker -d -H fd:// --registry-mirror=https://docker.mirrors.ustc.edu.cn
新版的 Docker 推荐使用 json 配置文件的方式，默认为 /etc/docker/daemon.json，非默认路径需要修改 dockerd 的 –config-file，在该文件中加入如下内容：
{"registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]}
